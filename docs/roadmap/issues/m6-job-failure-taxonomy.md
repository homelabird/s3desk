## 요약
Job 실패 유형 분류 체계를 정의하고, UI/로그/메트릭에 반영한다.

## 배경/문제
현재 실패 원인이 자유 텍스트 중심으로 남아 있어 운영/분석/재시도 판단이 어렵다.

## 목표
- 실패 유형 분류 체계(코드/범주)를 도입한다.
- UI와 로그에 일관된 실패 분류 정보를 제공한다.

## 범위
- 포함:
  - 실패 코드/범주 정의
  - 실패 코드 출력/저장
  - UI 표시 및 필터링
- 제외:
  - 외부 알림/자동 치유 정책

## 수용 기준 (Acceptance Criteria)
- [ ] 주요 실패 유형이 분류 코드로 기록된다.
- [ ] UI에서 실패 유형이 명확히 표시된다.
- [ ] 실패 유형별 통계 추출이 가능하다.

## 리스크/가정
- rclone 에러 포맷 변경 시 분류 로직 업데이트 필요

## 의존성
- Job 로그 구조 및 파서

## 테스트/검증
- 실패 유형별 샘플 로그 파싱 테스트
- UI 표시 검증

## 롤아웃/롤백
- 실패 분류 미지원 시 기본값 처리

## 메트릭
- 실패 유형별 비율
- 분류 불가 비율
