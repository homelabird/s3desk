## 요약
M8 즐겨찾기/북마크 MVP(단일 사용자) 정책/모델/API/UX를 확정하고 구현한다.

## 배경/문제
반복 탐색과 검색 비용이 커서, 자주 쓰는 객체/경로를 빠르게 재사용할 수 있는 경량 개인화가 필요하다.

## 목표
- 즐겨찾기(객체/프리픽스/버킷) 기본 UX 제공
- 저장된 검색(북마크) MVP 제공
- 데이터 모델/키 정규화/충돌 규칙 확정
- API 계약 및 기본 메트릭 확정

## 범위
- 포함:
  - 즐겨찾기 토글, 즐겨찾기 전용 필터/정렬
  - 북마크 저장/호출/삭제
  - 단일 사용자 범위 스코프
  - 즐겨찾기/북마크 API 및 최소 스키마
  - 기본 지표(토글 지연, 실패율)
- 제외:
  - 팀 공유 즐겨찾기/북마크
  - 자동 추천/컬렉션/태그
  - 교차 인스턴스 동기화

## 수용 기준 (Acceptance Criteria)
- [ ] 객체/프리픽스/버킷 즐겨찾기 토글이 정상 동작
- [ ] 즐겨찾기만 보기 + 즐겨찾기 먼저 정렬 제공
- [ ] 북마크 저장/호출/삭제가 가능
- [ ] prefix 키 정규화 규칙과 유니크 키 구성 확정
- [ ] 409(중복), 404(없음), 403(권한) 규칙 정의
- [ ] 토글 p50 < 200ms, p95 < 800ms 목표 설정
- [ ] 기본 UI 회귀 테스트 추가

## 리스크/가정
- 사용자 식별(user_id)이 일관되게 전달된다고 가정
- 대용량 목록에서 즐겨찾기 동기화 지연 가능성

## 의존성
- 사용자 식별 및 인증 컨텍스트
- 즐겨찾기/북마크 데이터 저장 스키마

## 테스트/검증
- API CRUD 테스트
- UI 토글/필터/정렬 회귀 테스트
- 대용량 목록 성능 영향 점검

## 롤아웃/롤백
- 기능 플래그로 점진 노출
- API는 버전 범위 내 하위 호환 유지

## 메트릭
- 토글 지연 시간(p50/p95)
- 토글 실패율
- 즐겨찾기/북마크 재사용률
