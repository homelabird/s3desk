## 요약
M9 팀/프로젝트 공유 즐겨찾기/북마크를 제공하고 권한/감사 로그와 연동한다.

## 배경/문제
원격/팀 운영 환경에서는 개인 즐겨찾기만으로는 협업 맥락을 공유하기 어렵다.

## 목표
- 팀/프로젝트 범위 즐겨찾기/북마크 제공
- 공유 정책과 권한 모델(RBAC) 연동
- 감사 로그에 공유 변경 이력 기록

## 범위
- 포함:
  - 팀 공유 즐겨찾기/북마크
  - 팀/프로젝트 스코프 분리
  - 공유 항목 권한 검사 및 감사 로그 연동
  - 공유 항목 UI 표시/필터
- 제외:
  - 외부 사용자 공유 링크
  - 크로스 인스턴스 동기화

## 수용 기준 (Acceptance Criteria)
- [ ] 팀 스코프에서 즐겨찾기/북마크 공유 가능
- [ ] 권한 없는 사용자는 공유 항목 접근 불가
- [ ] 공유 변경 이력이 감사 로그에 기록
- [ ] 개인/팀 즐겨찾기 분리 표시 및 필터 제공
- [ ] 공유 항목 삭제/권한 변경 시 UX 규칙 확정

## 리스크/가정
- 팀/프로젝트 모델과 권한 체계가 이미 확정되어 있다고 가정

## 의존성
- M9 인증/권한/RBAC 설계
- 감사 로그 파이프라인

## 테스트/검증
- 권한/스코프 테스트
- 감사 로그 기록 검증
- 공유 항목 UI 회귀 테스트

## 롤아웃/롤백
- 스코프 단위 플래그 적용
- 공유 기능 비활성화 시 개인 즐겨찾기는 유지

## 메트릭
- 공유 즐겨찾기/북마크 생성률
- 공유 항목 접근 성공/실패율
