## 요약
Job 로그/상태 보존 및 정리 정책을 정의하고 구현한다.

## 배경/문제
Job 로그가 무제한으로 쌓이거나, 정리 정책이 불명확해 운영 시 비용과 성능에 영향을 줄 수 있다.

## 목표
- 보존 기간/용량 상한을 정의한다.
- 자동 정리 정책과 동작을 문서화한다.

## 범위
- 포함:
  - 보존 기간/용량 상한 정책
  - 자동 정리 작업 및 UI 표시
  - 설정 옵션 추가 (JOB_LOG_RETENTION, JOB_LOG_MAX_BYTES, JOB_RETENTION)
- 제외:
  - 외부 로그 저장소 연동

## 수용 기준 (Acceptance Criteria)
- [x] 로그 보존 정책이 문서화되어 있다.
- [x] 보존 기간/용량 초과 시 자동 정리된다.
- [x] 사용자가 보존 정책을 확인할 수 있다.

## 리스크/가정
- 로그 삭제 시 디버깅 정보 손실 가능성

## 의존성
- Job 로그 저장 구조

## 테스트/검증
- 로그 정리 정책 시뮬레이션
- 보존 기간/용량 테스트

## 롤아웃/롤백
- 기본값은 보수적으로 설정
- 정책 변경은 설정값으로 롤백 가능

## 메트릭
- 로그 디스크 사용량 추이
- 정리 작업 성공률
